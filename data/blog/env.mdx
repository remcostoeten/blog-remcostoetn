## Introduction

Navigating the world of calling environment variables on the frontend in Next.js can be tricky, especially when you're prone to forget the nuances. In this article, we'll break down the process step by step, ensuring that you can confidently handle environment variables and optimize your Next.js application.

## Setting Up Environment Variables

To start, create a `.env` file in the root of your project and ensure it's added to the `.gitignore` file. This ensures sensitive information is not exposed in your version control.

```bash
# Create a .env file
touch .env
```

When deploying to Netlify, consider using `.env.local` or `.env.production` for different testing environments.

## Using Environment Variables in Next.js

In your `.env` file, assign variables to secret values. For instance, let's use a Google Firebase example:

```bash
KEY=AIzaSyCR3sqaL1zGtKPMMo9_r6d7WxLH9u3m2bY
AUTH_DOMAIN=purcasche-p.firebaseapp.com
PROJECTID=purcasche-p
BUCKET=570643970608
APP_ID=1:570643970608:web:a90a71d4a7e8e170f00bf5
```

In your `firebase.tsx` file, call these variables as follows:

```jsx
apiKey: process.env.NEXT_PUBLIC_FIREBASE_API_KEY,
authDomain: process.env.NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN,
projectId: process.env.NEXT_PUBLIC_FIREBASE_PROJECT_ID,
storageBucket: process.env.NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET,
messagingSenderId: process.env.NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID,
appId: process.env.NEXT_PUBLIC_FIREBASE_APP_ID,
```

Here, `NEXT_PUBLIC_` is a Next.js syntax, and the rest are the variables.
